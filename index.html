A hands-on tutorial

Grokking Fonts & Files

Understand how a font file is built, byte by byte — from the binary container that wraps it, through glyphs, metrics and shaping, up to font collections and variable fonts. Every fact is pulled from two real fonts on disk, so nothing here is hand-wavy.

The specimens on the bench We dissect LXGW WenKai Mono GB and LXGW ZhenKai GB (and the hybrid .ttc we assemble from them along the way). Reproduce any number in the lessons with specimens/extract.py <font>. See specimens/SPECIMENS.md for the full verified dataset.

How to use this

  • Each module has three parts: concept walkthrough on the real bytes, and exercises.
  • Exercises are concept questions or programming tasks. Attempt them, then paste your answer/output back to me (your tutor) — I'll check them. Some include a collapsible hint or self-check solution.
  • Click Mark module complete at the bottom of a page to track progress (stored in your browser).
Setup (optional but recommended) To run the coding exercises, have the two source fonts handy and a Python with fonttools. The repo bundles both fonts in specimens/fonts/ (via Git LFS); you can also fetch them from the LXGW GitHub releases. Many early exercises need nothing but xxd and a hex calculator.
