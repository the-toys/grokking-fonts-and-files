Module 03

Outlines

A glyph isn't a picture — it's a set of closed paths the renderer fills. In this font those paths are made of quadratic Bézier curves stored in glyf and located through loca. Learn how points become curves, how curves become a filled shape, and how one glyph can be assembled from others.

concept

How a glyph is drawn

A glyph outline is one or more closed contours — loops of connected points. The renderer doesn't draw the loops as lines; it fills the region they enclose. When contours nest (the hole inside an 'O', the counter inside '永'), a winding rule decides what counts as inside: non-zero winding (the usual TrueType/PostScript rule) or even-odd. Reverse the direction of an inner contour and, under non-zero winding, it punches a hole instead of adding ink.

Every coordinate lives in font units on the em square. For this font unitsPerEm = 1000, so a point like (110, 383) means 110 units right, 383 units up, on a 1000×1000 design grid. (The em and how it scales to pixels is the whole of Module 4 — for now just read coordinates as positions on a 1000-unit grid.)

Two outline technologies

There are two dominant ways to encode those curves, and which one a font uses is announced by its sfnt magic back in Module 1:

  • TrueType (what this font is — magic 00 01 00 00): uses quadratic Bézier curves. Outlines live in the glyf table, and each glyph's byte range is found through the loca index. Points carry a flag marking each as on-curve (an anchor the path actually passes through) or off-curve (a quadratic control point that bends the curve toward it without being touched).
  • CFF / CFF2 (PostScript-flavoured OTF — magic OTTO): uses cubic Bézier curves, stored compactly in a CFF  table (note the trailing space — still a 4-byte tag). No glyf/loca at all.
Quadratic vs cubic, in one line A quadratic segment has one control point; a cubic segment has two. Cubics can express an S-curve in a single segment, so CFF often needs fewer points; TrueType makes up for it with cheaper math and the "implied on-curve point" trick below. Same shapes, two encodings.

The implied on-curve point (TrueType only)

TrueType allows a shortcut that trips up first-time readers: when two off-curve control points sit next to each other, an on-curve point is implied at their exact midpoint. The font stores fewer anchors and the renderer reconstructs the missing ones. So a run of off-curve points isn't one impossible mega-curve — it's a chain of quadratic segments joined at implied midpoints.

loca: the glyph index into glyf

loca is to glyf what the table directory was to the whole file: an index of byte offsets. Entry i and entry i+1 give the start and end of glyph i's data inside glyf. That's why loca holds numGlyphs + 1 entries — you always need the offset after the last glyph to bound it. A side effect: if entry i equals entry i+1, glyph i has zero bytes — an empty glyph (like a space). The field head.indexToLocFormat selects the entry size: 0 = 16-bit "short" offsets (stored as the real offset ÷ 2), 1 = 32-bit "long" offsets stored directly.

Composite glyphs

A glyph need not draw any outline of its own. A composite glyph is built from references to other glyphs (its components), each placed with an offset and optionally a scale or 2×2 transform. The 'Ä' glyph can be "draw glyph 'A', then draw the dieresis glyph shifted up". This is enormous for:

  • CJK, where thousands of characters share radicals — draw the radical once, reference it everywhere.
  • Accented Latin, where a base letter plus a combining mark composes hundreds of precomposed characters.

It saves space (one stored outline, many references) and enforces visual consistency (every accent is the literal same shape).

Hinting, briefly At tiny pixel sizes a curve can fall awkwardly across the grid and a stem can blur or vanish. The prep and fpgm programs (TrueType bytecode) plus the gasp table control grid-fitting — nudging points onto pixel boundaries so stems stay crisp. WenKai's prep is only 7 bytes (Module 1's directory), and modern greyscale/antialiased rendering leans on hinting far less than 1990s black-and-white displays did. Useful to know it exists; don't overstate its role today.
walkthrough

Reading real outlines from WenKai

First, the format question. This font's head.indexToLocFormat = 1, so loca here uses 32-bit long offsets — each entry is a direct byte offset into glyf, no ÷2. With glyf weighing 24.7 MB (Module 1), 16-bit offsets couldn't reach far enough anyway.

A simple glyph: '一' (U+4E00)

The CJK character for "one" — a single horizontal stroke. fontTools reports it as isComposite=False, numberOfContours=1, 21 points, with endPtsOfContours=[20] (the one contour ends at point index 20, i.e. it owns points 0–20). Here are its first six points:

point:   0        1        2        3        4        5
coord:  (110,383)(837,420)(861,422)(881,430)(893,430)(921,413)
onCurve:  1        1        0        0        0        0
          └anchor  └anchor  └────── off-curve run ──────┘

Walk it as the renderer would, starting at point 0:

  • Point 0 (110,383), on-curve — the path starts here. The pen is placed on this exact coordinate.
  • Point 1 (837,420), on-curve — also an anchor. Two consecutive on-curve points mean a straight line from 0 to 1 (no control point between them to bend it). That's the long top edge of the stroke.
  • Points 2–5, all off-curve — a run of four control points (861,422) (881,430) (893,430) (921,413). Because consecutive off-curve points have implied on-curve points at their midpoints, this run is really several quadratic segments chained together: from anchor 1 the curve flows toward control 2, passes through the implied midpoint of 2&3, then toward control 3, through the midpoint of 3&4, and so on — rounding the right end of the stroke. The path never touches points 2–5; they only pull the curve.

So the flag string 1 1 0 0 0 0 reads as "two anchors, then a run of control points", and the key lesson is that a run of consecutive off-curve points carries implied on-curve points between each pair. The remaining 15 points (6–20) continue the loop back around and close it.

A second simple glyph: '十' (U+5341)

"Ten" — a vertical stroke crossing a horizontal one. One contour, 46 points, and its on-curve flags begin 1 1 0 1 0 1. Notice the alternating anchor/control rhythm (1 0 1 0…) after the first pair — that's the signature of straighter strokes with single curves at the corners, versus '一''s longer off-curve run that rounds one broad end. Same encoding, different cadence of points reflecting the different shape.

Composite glyphs: spacing punctuation built by reference

The standalone, spacing dieresis (the ¨ you type on its own, not the combining one) is a composite with 1 component: it references the combining-mark glyph dieresiscomb placed at (x=500, y=0). Likewise the spacing acute is 1 component referencing acutecomb at (x=517, y=0).

Read that literally: the dieresis glyph contains no outline of its own. Its entire glyf record is "draw the dieresiscomb outline, shifted 500 units right". The combining mark is designed to sit over a preceding letter (offset near zero), so the spacing version just translates it into the middle of its own cell. One outline, two characters, guaranteed identical shape — the composite mechanism in miniature.

See it for yourself ~/.venv/bin/python -c "from fontTools.ttLib import TTFont; f=TTFont('LXGWWenKaiMonoGB-Regular.ttf'); g=f['glyf']['u4E00']; print(g.numberOfContours, g.coordinates[:6], list(g.flags[:6]))" prints the contour count, the first coordinates, and the on-curve flags shown above (flag bit 0 = on-curve).

3.1concept

Classify the points of '一'. Given the flag sequence 1 1 0 0 0 0 for the first six points, label each point 0–5 as anchor (on-curve) or control (off-curve). Then explain what the two consecutive off-curve points 2 and 3 imply about the path between them.

Show solution Point 0 = anchor, 1 = anchor, 2 = control, 3 = control, 4 = control, 5 = control. Because points 2 and 3 are both off-curve and adjacent, TrueType implies an on-curve point at their midpoint: ((861+881)/2, (422+430)/2) = (871, 426). So the curve passes through that implied midpoint, splitting the 2→3 stretch into two quadratic segments rather than one impossible two-control curve.

TrueType vs CFF. For each technology state: how many control points per curve segment, which table(s) hold the outlines, and which sfnt magic (from Module 1) signals it.

Show solution TrueType: quadratic = 1 control point per segment; outlines in glyf indexed by loca; magic 00 01 00 00. CFF/CFF2: cubic = 2 control points per segment; outlines in the CFF  table (no glyf/loca); magic OTTO.

Inspect '十'. With fontTools, load the font, fetch the glyph 'u5341', and print its numberOfContours, its total number of points, and the count of on-curve vs off-curve points.

from fontTools.ttLib import TTFont
f = TTFont("LXGWWenKaiMonoGB-Regular.ttf")
g = f["glyf"]["u5341"]
n = len(g.coordinates)
on = sum(1 for fl in g.flags if fl & 1)
print(g.numberOfContours, n, on, n - on)

3.5concept

Count the loca entries. loca holds numGlyphs + 1 entries. This font's numGlyphs = 46830. How many loca entries are there, and why the extra one?

Show solution 46831 entries. Each glyph i's data spans from loca[i] to loca[i+1], so you need one offset past the end of the last glyph to give glyph 46829 an upper bound. Hence numGlyphs+1.
3.6concept

Why composites matter. Why are composite glyphs especially valuable for CJK fonts and for accented Latin? Name the two things they save.

Show solution They save space and visual consistency. CJK characters share radicals and accented Latin characters share base letters and combining marks; storing each shared piece once and referencing it (with a placement offset) avoids duplicating thousands of outlines, and because every reference points at the same outline, the radical or accent is guaranteed to look identical everywhere it appears.
