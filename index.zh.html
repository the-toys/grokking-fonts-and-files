动手实践教程

吃透字体文件

一个字节一个字节地弄清楚字体文件是怎么造出来的——从包住一切的二进制容器， 到字形、度量、整形，再到字体集合与可变字体。每个结论都取自磁盘上两只真实的字体文件， 没有一处含糊带过。

工作台上的标本 我们解剖 霞鹜文楷等宽 GB（LXGW WenKai Mono GB）和 霞鹜臻楷 GB（LXGW ZhenKai GB），以及一路上用它们拼装出来的混合 .ttc。课程里的任何数字都可以用 specimens/extract.py <font> 复现；完整的核验数据集见 specimens/SPECIMENS.md。

模块一览

01
sfnt 容器
字体文件是怎么摆放的？头部 + 表目录 + 偏移。
 02
cmap 与字形 ID
字符码如何变成字形。编码与各种子表格式。
 03
轮廓
字形是怎么画出来的——二次 glyf 与三次 CFF、复合字形、hinting。
 04
度量与 em
前进宽度、侧边距、基线。为什么全角汉字恰好宽一 em。
 05
身份与家族
name / OS/2 / head——操作系统怎么把几份文件归成一个家族。
 06
整形与布局
GSUB / GPOS、OpenType 特性、连字、HarfBuzz。
 07
打包与前沿
TTC 与共享表、子集化、可变字体与彩色字体、WOFF2。

怎么用这份教程

  • 每个模块分三部分：概念 解读（对着真实字节逐段拆解），以及 练习。
  • 练习分概念题和编程题。先自己做， 再把答案或输出贴给你的 AI 导师核对。部分练习附有可折叠的提示或参考答案。
  • 点击页面底部的 标记本模块完成 记录进度（存在你的浏览器里）。
环境准备（可选，但建议） 要做编程练习，手边得有这两只源字体，外加装了 fonttools 的 Python。 仓库已在 specimens/fonts/ 里带上两只字体（Git LFS）；也可以从 LXGW 的 GitHub releases 下载。前几个模块的不少练习只需要 xxd 和一个十六进制计算器。
